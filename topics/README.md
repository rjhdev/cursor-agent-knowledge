# Topic files

Each file in this directory covers one subject agents may need across projects.

## Naming

- Use lowercase with hyphens: `deployment.md`, `ci-pipelines.md`
- One primary topic per file; split large subjects into linked files
- Keep filenames short and descriptive

## File template

```markdown
# Topic title

Brief summary of what this file covers.

## Key facts

- Fact or convention agents should remember

## Common issues

- Problem → solution or workaround

## References

- [Link to authoritative docs](https://example.com)
```

## After creating a file

1. Add a row to [INDEX.md](../INDEX.md)
2. Open a pull request with a clear summary of what was added and why
